A) Setting up web server - top

Setting up a web server is just like placing web contents on a web server. You've to copy the web server folder from project directory and place it on your web server. You should set certain attributes on your web server to avoid leakage of your server files. After adding files, you should make changes to your config.php file which is located inside /include/ folder.

This is the half configuration file data which needs to be changed before you set your app up. You should change DB_USERNAME, DB_PASSWORD, DB_HOST and DB_NAME with your database username, password, name and database host. If you're hosting database on the same machine then use localhost as the default host of database.

As for the HOST and PROFILE_HOST replace it with your machine host (ip address). Make sure to replace <'your-hostip-address'> in PROFILE_HOST instead of whole link as it is used to load profile pictures from the server.

Now you've to add your FCM (Firebase Cloud Messaging) server key on variable FCM by replacing " <'fcm-id'>". Your server configuration is updated and server is working now. Now follow next steps to get the app working.

B) Setting up admin panel - top

Setting up an admin panel is just like setting up a web server. You just have to place admin panel files on your web server folder. After placing the files, you've to execute some database queries to make the admin panel work and for login credentials. You can find database file inside /Admin Panel/database files inside project folder. You've to import the database file or either execute the queries on your database. After executing queries, at last, we've to change configuration. For this you've to edit configuration file inside /core/db_config.php.

This is the configuration file for admin panel to work with the social app. You should change user, pass, name, env. user and pass is for database credentials and 'name' is for database name same as on config file. You've to provide the same credentials as you gave on config.php file for database connection.

'env' is for changing the property of admin panel. If it's for demo then make it 0 or if it is for production make it 1.

That's all. If you encounter any issue on this, be sure to contact me.

After setting up server configuration and connecting the database with it. You now need to setup the app and set its configuration to connect with the server. The app configuration is so simply and you only need to do two things. First thing, you've to add google-services.json file inside the app folder which will make the notification and messaging work. You can view on how to do this below.

Second thing is setting up the configuration according to server configurations so that app can connect on it. After building your project, goto config.java file which is located inside App > src > main > java > config > config.php . The file will look something like this:

C) Setting up FCM (Firebase Cloud Messaging) - top

Firebase Cloud Messaging (FCM) is used for:

  1. Push Notifications
  2. Messages
  3. Deal with certain background syncing

You need to have a google account to setup FCM. After creating a google account, goto Firebase Console and click the Create New Project button.

Name your project and click the Create Project button.

Now click Add Firebase to your Android app button and enter your app package name (default is: innovativedeveloper.com.socialapp) and click the Add App button. Follow the instructions on-screen to move the downloaded google-services.json file into your app. (instructions also given above) Click the Continue button. Now retrieve the server api key which is important to be placed in config.php file on server-side. You can do this by clicking the overflow button on your project and clicking the Manage item. Select the Cloud Messaging tab and note the server key or directly paste it on config.php file in server. (instructions are also given here )

D) Usage of libraries - top

These are the following libraries used in this app for certain features:

  1. de.hdodenhof:circleimageview : It is used for round images and profile views.
  2. com.squareup.picasso : It is used to load images, snapshots and uploading images.
  3. 'com.github.danylovolokh:hashtag-helper : It is used for hashtag feature within the app.
  4. com.klinkerapps:link_builder : It is used to build links within a textview for double profile links.
  5. com.balysv:material-ripple : Material ripple is used for ripple effects for buttons and controls to provide good ripple effect.
  6. jp.wasabeef:blurry : It is used to add blur background on profile view.
  7. com.rockerhieu.emojicon : For emoticons.

Q) I am not able to upload large files (i.e videos, photos)?
A) You've to view your php settings using phpinfo() snippet. Search for upload_max_filesize and post_max_size and check whether they're set at what you want to be the upload size of files.
